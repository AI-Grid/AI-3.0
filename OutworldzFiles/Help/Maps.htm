Maps

Opensim has many different maps settings. DreamGrid has multiple easy-to-use combinations.   Opensimulator has several Map tile makers which vary in quality, and speed.  It can take an exceedingly long time to boot when using the Better and Best maps.  These maps which use the Warp3D engine must load all prims, mesh, and textures, so it takes a long time to generate new maps.  Also, any bad textures may cause harmless errors from corrupted assets to appear on your console.

I recommend you run all maps at Best setting once, then set Maps to None.  You can remake maps when enough changes have been made.

·        None:  No maps will be made. This is a good setting as the regions will boot very quickly.  Any existing maps are not deleted.

·        Simple but Fast: MapImageModule is used with just Land showing

·        Good:  Uses Warp3D module with just Land showing

·        Better: Uses Warp3D module with Land, Prims, and land Textures showing

·        Best: Uses Warp3D module with Land, Prims, Mesh, Sculpts, and all Textures including prims showing.

·        Delete All Maps:  If you delete a region, the map will remain. Click this to clear out all maps.  You will need to regenerate all maps again by choosing a setting and restarting all regions.

·        View Map Web page: Enabled if your system is booted.  Opens a web browser to http://localhost:8002/wifi/map.htm.

·        Small Metro Map and Large Metro Maps Available only if your Apache/PHP is enabled in Settings Web Pages. These maps are generated by PHP and Apache web server.



Manual regeneration

Normally you would generate map tiles at startup. You can also manually force map tile regeneration with the console command 'generate map<enter>'.

Stop the system, but do not exit. The first step after it has stopped is to go to Setup->Settings->Regions, where the Global Maps are set for all regions.

 

There is a "deregister all regions" button. 

If you typed 'q' or 'quit' or pressed stop, they are deregistered from the system.  Sometimes a region crashes and remains behind which can confuse some software systems. Maps need to be cleared and rebuilt  when you delete a region or move them around.

Example 1:  You get an error when you load a new region with the same name, or try to use the same map location. 

Example 2: you moved a region, and it appears twice on the maps.

Example 3: you deleted a region, and it still appears on the map.

Simplest way to fix this begins with stopping the system with the stop button.  You need to do this first as we are going to clear out all regions, and let the system build it when it boots.

 

Click the button.  This removes all regions from the database.  Regions will re-register when the region  starts. 

Step 2 is to remake the maps. 

A) Go to Setup Settings->Maps and erase them all with "Delete All Maps"

B) Now choose "Best (prims + Mesh Very Slow)"

C) Close this window which saves the new Global setting.

D) Press Start.

E) Once all the regions have booted to the DOS boxes, set map back to "None".  They can still be making maps. They just need to all be starting up and then you can change the Global Settings. 

The next time you boot, the regions will load very quickly as the maps will not be remade. Maps only need to be cleared when you delete a region or move them around.

F) Each region has an override setting.   If set, the override will choose a different setting for maps.

Example: I leave the global maps off all the time.   But I may be working heavily on a region and changing it around, and want that to appear after every reboot.  I go to the Regions List and click the Region and set the override to the desired setting, and reboot the region.

Select Edit:

Then choose a Map setting and save the Region.
